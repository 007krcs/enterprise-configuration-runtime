# Quick Start

Follow this path to go from zero to a governed bundle in under five minutes.

## 1. Load an example bundle

Visit `/examples`, pick **E-Commerce Store**, **SaaS Dashboard**, or **Onboarding Journey**, and click **Open in Builder**.
The empty canvas is replaced with the sample UI schema + flow + rules so you can experiment immediately.

## 2. Customize a screen

Switch to the canvas, drag a table or card from the palette, then move it and tweak props via the inspector.
Use the toolbar to toggle rulers or change artboard breakpoints so you can see how the interface adapts.

## 3. Guard a transition with rules

Open **Rules** (`/builder/rules`), add a rule, set the JSON guard (for example `cart.total > 0`), and save.
This rule is wired to a flow transition so the runtime will evaluate it before moving to the next state.

## 4. Preview the runtime

Switch to **Playground** (`/playground`), change a bound input (`orderTotal`, `user.role`, etc.), and click **Submit** or **Next**.
The trace reveals each flow transition and every rule evaluation.

## 5. Export the JSON

In **Console** (`/console?tab=versions`), click **GitOps > Export** to download the bundle you just edited.
Use the exported file to seed another tenant, diff against a merged change, or commit to GitOps storage.
